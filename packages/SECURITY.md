# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, **do not open a public GitHub Issue**.

Please report it privately by emailing:

**security@kagura.run**

Include as much detail as possible:
- A description of the issue and potential impact
- Steps to reproduce (proof of concept if available)
- Affected components/versions

## Response Time

We aim to:
- acknowledge reports within **72 hours**
- provide a remediation plan or status update within **7 days**

## Disclosure Policy

We follow responsible disclosure:
- Please allow reasonable time for investigation and a fix before public disclosure
- We may coordinate a release and advisory with the reporter
- Credit will be given to the reporter if desired
